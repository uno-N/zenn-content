# Note 記事作成ガイド

## 読者像
- 医療×テックに関心がある人
- プログラミング未経験〜初心者
- 音楽療法関係者
- 個人開発者
- ゆーのさんの成長を応援している人

## Noteで求められること
- **共感**：「わかる！」と思ってもらえる失敗談や感情
- **ストーリー性**：なぜやったか・何が起きたかの流れ
- **わかりやすさ**：専門用語を自分の言葉で噛み砕く
- **親しみやすさ**：医療者としての視点・例えを活かす

## 📝 記事執筆前（企画段階）

### テーマ・目的

- [ ] この記事で伝えたい**メインメッセージ**は明確か？
- [ ] **読者ペルソナ**（誰に向けて書くか）を決めたか？
    - □ 医療×テックに関心がある人
    - □ React初学者
    - □ 音楽療法関係者
    - □ 個人開発者
- [ ] 読者がこの記事から得られる**価値**は何か？
    - □ 知識・情報
    - □ 共感・励まし
    - □ 具体的な手順・ノウハウ

### キーワード設定

- [ ] 検索されやすいキーワードを3-5個リストアップしたか？
    - 例）音楽療法、看護師、React、個人開発、アプリ開発
- [ ] そのキーワードでGoogle検索して、競合記事を確認したか？

## ✍️ 執筆中

### タイトル

- [ ] **内容が想像できる**具体的なタイトルか？
    - ❌ 「開発日記 #1」
    - ⭕ 「看護師がReactでアプリ開発を始めた理由と最初の壁」
- [ ] **キーワード**がタイトルに入っているか？
- [ ] 30-40文字程度に収まっているか？
- [ ] 複数のタイトル案を比較したか？
    - 案1: __________________
    - 案2: __________________
    - 案3: __________________

### 構成

- [ ] **逆三角形**（結論→説明→補足）の構成になっているか？
- [ ] 冒頭で**この記事で得られること**を明示したか？
- [ ] 長い記事は**目次**を入れたか？（1500文字以上推奨）
- [ ] 見出しだけ読んで流れがわかるか確認したか？

**記事の型（参考）**
```
背景・動機（なぜやろうとしたか）
  ↓
やってみた（何が起きたか・失敗も含めて）
  ↓
学んだこと
  ↓
次回予告
```

### 本文

- [ ] **スマホで読みやすい**改行になっているか？（2-3文ごとに改行）
- [ ] **1文を短く**（50-60文字以内）したか？
- [ ] 専門用語は**簡単な言葉**で説明したか？
- [ ] 抽象的な内容に**具体例**を添えたか？（医療の例えも活用）
- [ ] 漢字が続きすぎていないか確認したか？
    - 例）「出来る」→「できる」、「有る」→「ある」
- [ ] 失敗・つまづきを隠さず書いているか？（それが一番刺さる）

### 視覚要素

- [ ] **スクリーンショット・図**を適切に配置したか？（3-5段落ごとに1枚）
- [ ] 画像の**ALTテキスト**を設定したか？
- [ ] 箇条書き・引用機能を活用したか？
- [ ] コードブロックは適切にフォーマットしたか？（技術記事の場合）

### 引用・参照

- [ ] 参照した情報源に**リンク**を貼ったか？
- [ ] 引用する場合は**出典を明記**したか？

## 💡 Tips

### 筆が進まないとき
- 完璧を目指さない（80%で公開して反応を見る）
- 過去の自分に向けて書く
- 音声入力で下書きを作る

### ネタに困ったとき
- 開発中につまずいたことを記事にする
- 1週間前の自分が知りたかったことを書く
- 読者からの質問に答える

### 継続のコツ
- 執筆日・時間を固定する
- 下書きストックを3本持つ
- 完璧主義を捨てる
