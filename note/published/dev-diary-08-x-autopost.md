# 非エンジニア看護師がX自動投稿システムを1から作った話｜情報収集から引用ポストまで全自動にした

「興味ある分野のXポストを、毎日自動でメールに送ってもらえる機能がある」

X（Twitter）に、そんな機能があることを知った。

Grok/Tasksという機能で、キーワードを設定しておくと、
関連するXのトレンドや注目ポストをメールで届けてくれる。
（設定はここから → https://grok.com/tasks ）

これを見て思った。

**「このメールを読んで、引用リツイートを自動で投稿する仕組みが作れるんじゃないか」**

情報収集も、コメントを考えるのも、投稿するのも、全部自動にしたかった。

---

## 作ったものの全体像

仕組みはこんな流れ。

```
Grok/Tasks
  ↓ 興味ある分野のポストをメールで送信
Gmailに届く
  ↓ PythonでIMAP接続して取得
元ポストのURLとIDを抽出
  ↓ Claude APIで引用コメントを生成
Xに引用ポストとして投稿
```

1日3回、自動で動く。

- 朝6時・夕18時 → Grokトレンドを引用ポスト
- 昼12時 → 個人開発・看護×ICT・映画音楽療法の3カテゴリをローテーション投稿

毎日自分でポストを考えなくていい。
でも、自分の声で発信できる。

そこが大事だった。

## 一番の問い：「自分の声」って自動化できるの？

Claudeに自分の声を学習させる部分、これが一番難しかった。

SYSTEM_PROMPTに詰め込んだものはある。

- 訪問看護師10年目・個人開発者としての人物像
- 「情景にコンテキストを込める」文体の方針
- 過去に伸びた投稿パターン3つ
- 看護×ICTで書いてはいけないこと（患者情報をAIに入力しているような表現など）

生成されるものは悪くない。こんな文章も出てきた。

```
テクノロジーは、在宅ケアを「効率化」するためにあるんじゃないと思ってる。

訪問の記録を素早く終わらせたいのは、その分だけ玄関先で立ち話できるためで。

「患者との時間」は、余りじゃなくて、本来の全部なので。
```

でも正直に言うと、**これを自動でそのまま投稿していいのか、まだ迷っている。**

自分のセンスで選んで投稿するなら、全自動は難しいと感じてる。
生成された内容は少しニュアンスが違うものばかりだった。

今は「叩き台を自動生成して、自分で確認してから投稿する」くらいが現実的かもしれない、という結論に近づいている。

## 一番大変だったのはここ

**X APIとアプリを繋ぐ部分**、これが本当に大変だった。

コードは書けた。
APIキーも取得した。
Gmailの連携も動いた。

でも、Xに投稿しようとすると毎回これが出る。

```
403 Forbidden
When authenticating requests to the Twitter API v2 endpoints,
you must use keys and tokens from a Twitter developer App
that is attached to a Project.
```

キーを再生成した。権限を確認した。
OAuth 2.0に切り替えた。何度も試した。

全部同じエラー。

---

## 解決のきっかけ

Geminiに相談したら、こんな回答をもらった。

「古いアプリがXの新しいAPI登録システムと不整合を起こしている可能性がある。新しいアプリを作り直すのが最短ルート。」

試しに、X Developer Console（ https://console.x.com ）で**アプリを新規作成**した。
そして、コードに `user_auth=True` を1行追加した。

```python
client.create_tweet(text=text, user_auth=True)
```

実行したら——

```
✅ 投稿完了！ tweet_id: 2044064982363750626
```

2日間格闘してたエラーが、アプリ作り直しとフラグひとつで消えた。

**API設定でつまずくときは、コードより先にアプリ設定を疑う。**

これが今回一番の学びだった。
