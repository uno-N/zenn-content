# Qiita 記事作成ガイド

## 読者像
- 国内エンジニア・開発者（最大規模コミュニティ）
- **検索で辿り着く人**が多い（Google・Qiita内検索）
- 「このエラーを解決したい」「この手順を知りたい」という目的がはっきりしている
- 初心者〜中級者が多い

## Qiitaで求められること
- **検索されやすいタイトル**：「〇〇 エラー 解決」「〇〇 使い方 入門」
- **再現性**：読者が同じ手順で試せること
- **完結性**：その記事だけで問題が解決できること
- **正確さ**：技術用語・コードに誤りがないこと

## ZennとQiitaの使い分け

| Zenn向き | Qiita向き |
|---------|---------|
| 開発日記・学習記録 | エラー解決法 |
| 個人の体験・考察 | 〇〇の使い方まとめ |
| 連載もの | 環境構築ハウツー |
| 「私はこう思った」 | 「こうすればできる」 |

### テーマ・目的

- [ ] この記事で**解決できる問題**は何か明確か？
- [ ] **読者ペルソナ**（誰に向けて書くか）を決めたか？
    - □ プログラミング初心者
    - □ 同じエラーで詰まっている人
    - □ 環境構築したい人
    - □ 特定の技術を学び始めた人
- [ ] 読者がこの記事から得られる**価値**は何か？
    - □ エラーの解決
    - □ 手順の再現
    - □ 概念の理解

### キーワード設定

- [ ] タグを1〜5個決めたか？
    - 例）React, Python, Node.js, 初心者, 環境構築

### ZennとのMarkdown差異

| 記法 | Qiita | Zenn |
|------|-------|------|
| 注意書き | `:::note warn` | `:::message alert` |
| 補足 | `:::note info` | `:::message` |
| 折りたたみ | `:::details` | `:::details`（同じ） |
| コードブロック | 同じ | 同じ |

## 💡 Tips

### Zennからの横流し手順
1. `articles/` のファイルをコピーして `qiita/drafts/` に置く
2. frontmatterを削除（Qiitaは不要）
3. Zenn独自記法をQiita記法に変換（`:::message` → `:::note info`）
4. タイトルに検索キーワードを追加
5. 冒頭に前提環境・バージョンを追記

### 筆が進まないとき
- 「自分がつまづいたこと」をそのまま書く
- まずコードブロックだけ並べて、説明を後から書く
- 500字の短い記事でもOK

### ネタに困ったとき
- 開発中に調べたことを記事にする
- エラーの解決方法をまとめる
- 「公式ドキュメントだけじゃわからなかったこと」を書く
